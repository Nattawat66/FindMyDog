บทที่ 5 การทดสอบระบบ

บทนี้นำเสนอผลการทดสอบในส่วนต่าง ๆ ของระบบ FindMyDog ครอบคลุมทุกฟังก์ชันการทำงานหลัก ได้แก่ ระบบเข้าสู่ระบบ, สมัครสมาชิก, ลงทะเบียนและจัดการสุนัข (CRUD), ระบบข่าวสาร (CRUD), โปรไฟล์ผู้ใช้, แผนที่สุนัขสูญหาย, ระบบค้นหาสุนัขด้วย AI และระบบขออุปการะสุนัข โดยแสดงรายการทดสอบ เงื่อนไข และผลที่ได้รับในแต่ละกรณี

5.1 การทดสอบในส่วนของผู้ใช้ทั่วไป

ตารางที่ 5.1: ผลการทดสอบระบบเข้าสู่ระบบ (Login)

การทดสอบเงื่อนไขในการทดสอบผลการทดสอบ
ทดสอบการเข้าสู่ระบบ ผู้ใช้เข้าหน้าเข้าสู่ระบบ (GET /login/) ระบบแสดงฟอร์มเข้าสู่ระบบ ผ่าน
ผู้ใช้กรอก Username และ Password ถูกต้อง แล้วกดเข้าสู่ระบบ ระบบตรวจสอบผ่าน authenticate() แล้ว redirect ไปหน้า Home ผ่าน
ผู้ใช้กรอก Username หรือ Password ผิด แล้วกดเข้าสู่ระบบ ระบบแสดง error "ชื่อผู้ใช้งานหรือรหัสผ่านไม่ถูกต้อง" ผ่าน
ผู้ใช้ไม่กรอกข้อมูลและกดเข้าสู่ระบบ ระบบแสดง error "กรุณากรอกชื่อผู้ใช้งานและรหัสผ่าน" ผ่าน
ผู้ใช้ Admin (is_staff=True) กรอกถูกต้องและเข้าสู่ระบบ ระบบ redirect ไปหน้า Admin Dashboard โดยตรง ผ่าน
ผู้ใช้ที่บัญชีถูกระงับ (is_active=False) พยายามเข้าสู่ระบบ ระบบแสดง error "บัญชีผู้ใช้งานนี้ถูกระงับการใช้งาน" ผ่าน
ผู้ใช้ที่ยังไม่ได้ Login พยายามเข้าหน้าที่ต้องการ Login เช่น /dog-list/ ระบบ redirect ไปหน้า Login พร้อม parameter ?next= ผ่าน

ตารางที่ 5.2: ผลการทดสอบระบบสมัครสมาชิก (Register)

การทดสอบเงื่อนไขในการทดสอบผลการทดสอบ
ทดสอบการสมัครสมาชิก ผู้ใช้เข้าหน้าสมัครสมาชิก (GET /register/) ระบบแสดงฟอร์มสมัครสมาชิก ผ่าน
ผู้ใช้กรอกข้อมูลครบถ้วนถูกต้อง (Username, Email, Password, Confirm, Phone, Line ID) และกดสมัคร ระบบสร้างบัญชีใหม่และ redirect ไปหน้า Login ผ่าน
ผู้ใช้กรอก Password ทั้งสองช่องไม่ตรงกัน และกดสมัคร ระบบแสดง error "รหัสผ่านไม่ตรงกัน" และไม่สร้างบัญชี ผ่าน
ผู้ใช้กรอก Username ที่มีอยู่แล้วในระบบ และกดสมัคร ระบบแสดง error "ชื่อผู้ใช้งานนี้มีอยู่ในระบบแล้ว" ผ่าน
ผู้ใช้กรอก Email ที่มีอยู่แล้วในระบบ และกดสมัคร ระบบแสดง error "อีเมลนี้มีอยู่ในระบบแล้ว" ผ่าน
ผู้ใช้ไม่กรอก Email (ไม่บังคับ) แต่กรอกข้อมูลอื่นครบ และกดสมัคร ระบบสร้างบัญชีโดยบันทึก Email เป็นค่าว่าง และ redirect ไปหน้า Login ผ่าน

ตารางที่ 5.3: ผลการทดสอบการลงทะเบียนสุนัข (Create)

การทดสอบเงื่อนไขในการทดสอบผลการทดสอบ
ทดสอบการลงทะเบียนสุนัข ผู้ใช้ที่ Login แล้วเข้าหน้าลงทะเบียนสุนัข (GET /register-dog/) ระบบแสดงฟอร์มลงทะเบียนสุนัขพร้อม FormSet รูปภาพ ผ่าน
ผู้ใช้กรอกข้อมูลสุนัขครบถ้วนพร้อมแนบรูปภาพ และกดบันทึก ระบบบันทึกสุนัข กำหนด owner เป็นผู้ใช้ปัจจุบัน ส่งรูปไป Embedding API แล้ว redirect ไปหน้ารายการสุนัข ผ่าน
ผู้ใช้ไม่กรอกชื่อสุนัข (ช่องบังคับ) และกดบันทึก ระบบแสดง error validation บนฟอร์มและไม่บันทึก ผ่าน
ผู้ใช้ที่ยังไม่ Login พยายามเข้าหน้าลงทะเบียนสุนัข ระบบ redirect ไปหน้า Login ผ่าน
ผู้ใช้ที่มีบทบาท Org Admin ลงทะเบียนสุนัข ระบบใช้ฟอร์ม OrgAdminDogForm ที่มีตัวเลือกพิเศษ (เช่น ฟิลด์ organization) และบันทึกสำเร็จ ผ่าน

ตารางที่ 5.4: ผลการทดสอบการดูรายละเอียดสุนัข (Read)

การทดสอบเงื่อนไขในการทดสอบผลการทดสอบ
ทดสอบการดูรายละเอียดสุนัข เจ้าของสุนัขเข้าหน้ารายละเอียดสุนัขของตน ระบบแสดงข้อมูลสุนัขครบถ้วน พร้อมปุ่มแก้ไขและลบ ผ่าน
ผู้ใช้ทั่วไปที่ไม่ใช่เจ้าของเข้าดูรายละเอียดสุนัขของคนอื่น ระบบแสดงข้อมูลสุนัขได้ แต่ไม่แสดงปุ่มแก้ไข/ลบ (can_edit=False) ผ่าน
Admin (is_staff=True) เข้าดูรายละเอียดสุนัขของผู้ใช้คนอื่น ระบบแสดงข้อมูลครบ และได้รับสิทธิ์แก้ไข (can_edit=True) ผ่าน
ผู้ใช้เรียก URL ที่ dog_id ไม่มีในระบบ เช่น /dogs/9999/ ระบบคืนหน้า 404 Not Found ผ่าน
ผู้ใช้เข้าหน้ารายการสุนัขทั้งหมด (Dog All List) ระบบแสดงรายการสุนัขทั้งหมดพร้อมสถิติ (จำนวน, สูญหาย, องค์กร) ผ่าน

ตารางที่ 5.5: ผลการทดสอบการแก้ไขข้อมูลสุนัข (Update)

การทดสอบเงื่อนไขในการทดสอบผลการทดสอบ
ทดสอบการแก้ไขข้อมูลสุนัข เจ้าของสุนัขเปิดโหมดแก้ไข (?edit=true) แก้ไขชื่อ และกดบันทึก ระบบบันทึกข้อมูลที่แก้ไขและ redirect กลับหน้า dog_detail ผ่าน
เจ้าของสุนัขเพิ่มรูปภาพใหม่ในโหมดแก้ไข และกดบันทึก ระบบบันทึกรูปใหม่ ส่งไป Embedding API และอัปเดต embedding_binary ผ่าน
เจ้าของสุนัขลบรูปภาพที่มีอยู่ในโหมดแก้ไข และกดบันทึก ระบบลบรูปภาพออกจากฐานข้อมูล ผ่าน
ผู้ใช้ที่ไม่ใช่เจ้าของส่ง POST request เพื่อแก้ไขสุนัขของคนอื่น ระบบแสดง error "คุณไม่มีสิทธิ์แก้ไขข้อมูลสุนัขตัวนี้" และ redirect กลับ ผ่าน
เจ้าของแก้ไขข้อมูลโดยล้างชื่อสุนัขให้ว่าง และกดบันทึก ระบบแสดง error validation และไม่บันทึกข้อมูล ผ่าน

ตารางที่ 5.6: ผลการทดสอบการลบข้อมูลสุนัข (Delete)

การทดสอบเงื่อนไขในการทดสอบผลการทดสอบ
ทดสอบการลบสุนัข เจ้าของสุนัขกดยืนยันการลบ (POST /dogs/<id>/delete/) สุนัขของตน ระบบลบสุนัขออกจากฐานข้อมูลและ redirect ไปหน้ารายการสุนัข ผ่าน
ผู้ใช้ที่ไม่ใช่เจ้าของส่ง POST request เพื่อลบสุนัขของคนอื่น ระบบคืน 404 Not Found เนื่องจาก get_object_or_404 กรอง owner ผ่าน
ผู้ใช้ส่ง GET request ไปที่ URL ลบสุนัข (ไม่ผ่าน Modal) ระบบ redirect กลับหน้า dog_detail โดยไม่ลบข้อมูล ผ่าน
ผู้ใช้ที่ยังไม่ Login พยายามลบสุนัข ระบบ redirect ไปหน้า Login ผ่าน

ตารางที่ 5.7: ผลการทดสอบการดูรายการสุนัขของตน (Dog List)

การทดสอบเงื่อนไขในการทดสอบผลการทดสอบ
ทดสอบการดูรายการสุนัข ผู้ใช้ทั่วไปเข้าหน้ารายการสุนัข (GET /dog-list/) ระบบแสดงเฉพาะสุนัขที่เป็นของผู้ใช้คนนั้น ผ่าน
Admin เข้าหน้ารายการสุนัข ระบบแสดงสุนัขทุกตัวในระบบ (ไม่จำกัด owner) ผ่าน
ผู้ใช้ที่ยังไม่มีสุนัขในระบบเข้าหน้ารายการสุนัข ระบบแสดงหน้าว่างพร้อมข้อความแนะนำให้เพิ่มสุนัข (has_dogs=False) ผ่าน

5.2 การทดสอบระบบข่าวสาร (Notification)

ตารางที่ 5.8: ผลการทดสอบการดูรายการข่าวสาร (Notification Read)

การทดสอบเงื่อนไขในการทดสอบผลการทดสอบ
ทดสอบการดูรายการข่าวสาร ผู้ใช้ทั่วไปเข้าหน้ารายการข่าวสาร ระบบแสดงข่าวสารประเภท ACTIVITY และ LOST_DOG ที่มองเห็นได้สาธารณะ ผ่าน
ผู้ใช้ที่เป็น AdoptionParent เข้าดูรายการข่าวสาร ระบบแสดงข่าวสาร DOG_SPECIFIC ที่เกี่ยวกับสุนัขที่ตนอุปการะด้วย ผ่าน
ผู้ใช้ Org Admin เข้าดูรายการข่าวสาร ระบบแสดงข่าวสาร DOG_SPECIFIC ของสุนัขในองค์กรทั้งหมด ผ่าน
ผู้ใช้คลิกข่าวสารเพื่อดูรายละเอียด ระบบโหลด Modal แสดงรายละเอียดข่าวสารพร้อมปุ่มแก้ไข/ลบตามสิทธิ์ ผ่าน

ตารางที่ 5.9: ผลการทดสอบการสร้างข่าวสาร (Notification Create)

การทดสอบเงื่อนไขในการทดสอบผลการทดสอบ
ทดสอบการสร้างข่าวสาร Org Admin เข้าหน้าสร้างข่าวสาร กรอกข้อมูลครบ และกดเผยแพร่ ระบบบันทึกข่าวสารโดยกำหนด organization เป็น Org Admin คนนั้น และ redirect ไปหน้ารายการ ผ่าน
Org Admin เลือกประเภทข่าวสาร DOG_SPECIFIC แต่ไม่ระบุสุนัข และกดเผยแพร่ ระบบแสดง error "ประกาศเฉพาะสุนัข ต้องระบุสุนัขที่เกี่ยวข้อง" ผ่าน
ผู้ใช้ทั่วไป (role=standard) พยายามเข้าหน้าสร้างข่าวสาร ระบบแสดง error "คุณไม่มีสิทธิ์ในการสร้างประกาศข่าวสาร" และ redirect ผ่าน
Org Admin แนบรูปภาพประกอบข่าวสาร และกดเผยแพร่ ระบบบันทึกรูปภาพและสร้างข่าวสารสำเร็จ ผ่าน

ตารางที่ 5.10: ผลการทดสอบการแก้ไขข่าวสาร (Notification Update)

การทดสอบเงื่อนไขในการทดสอบผลการทดสอบ
ทดสอบการแก้ไขข่าวสาร เจ้าของประกาศ (organization) แก้ไขข้อมูลและกดบันทึก ระบบบันทึกการแก้ไขและ redirect ไปหน้ารายการข่าวสาร ผ่าน
ผู้ใช้ที่ไม่ใช่เจ้าของและไม่ใช่ Admin พยายามแก้ไขประกาศของคนอื่น ระบบแสดง error "คุณไม่มีสิทธิ์ในการแก้ไขประกาศนี้" และ redirect ผ่าน
Admin (is_staff=True) แก้ไขประกาศของ Org Admin ระบบอนุญาตให้แก้ไขและบันทึกได้ ผ่าน

ตารางที่ 5.11: ผลการทดสอบการลบข่าวสาร (Notification Delete)

การทดสอบเงื่อนไขในการทดสอบผลการทดสอบ
ทดสอบการลบข่าวสาร เจ้าของประกาศส่ง POST request ลบประกาศของตน ระบบลบข่าวสารออกจากฐานข้อมูลและ redirect ไปหน้ารายการ ผ่าน
ผู้ใช้ที่ไม่ใช่เจ้าของพยายามลบประกาศของคนอื่น ระบบแสดง error "คุณไม่มีสิทธิ์ในการลบประกาศนี้" และ redirect ผ่าน
ผู้ใช้ส่ง GET request ไปที่ URL ลบข่าวสาร ระบบ redirect กลับหน้ารายการข่าวสารโดยไม่ลบ ผ่าน

5.3 การทดสอบระบบโปรไฟล์ผู้ใช้

ตารางที่ 5.12: ผลการทดสอบระบบโปรไฟล์ผู้ใช้

การทดสอบเงื่อนไขในการทดสอบผลการทดสอบ
ทดสอบโปรไฟล์ผู้ใช้ ผู้ใช้เข้าหน้าโปรไฟล์ตนเอง (GET /profile/) ระบบแสดงข้อมูลส่วนตัว (Username, Email, Phone, Line ID, Role) ในโหมดดูอย่างเดียว ผ่าน
ผู้ใช้เปิดโหมดแก้ไข (?edit=true) แก้ไข Phone และ Line ID แล้วกดบันทึก ระบบอัปเดตข้อมูลและแสดงข้อความ "บันทึกข้อมูลส่วนตัวเรียบร้อยแล้ว" ผ่าน
ผู้ใช้ส่ง POST ในโหมดดูอย่างเดียว (is_edit_mode=False) ระบบไม่บันทึกการเปลี่ยนแปลงและ redirect ไปหน้าโปรไฟล์ ผ่าน
ผู้ใช้ที่ยังไม่ Login พยายามเข้าหน้าโปรไฟล์ ระบบ redirect ไปหน้า Login ผ่าน

5.4 การทดสอบระบบแผนที่สุนัขสูญหาย

ตารางที่ 5.13: ผลการทดสอบระบบแผนที่สุนัขสูญหาย

การทดสอบเงื่อนไขในการทดสอบผลการทดสอบ
ทดสอบแผนที่สุนัขสูญหาย ผู้ใช้ที่ Login แล้วเข้าหน้าแผนที่สุนัขสูญหาย ระบบแสดงหน้าแผนที่พร้อม Widget แผนที่ ผ่าน
ระบบโหลดข้อมูล API สุนัขสูญหาย (/lost-dogs-map-data/) ระบบส่ง JSON ที่มีข้อมูล id, name, lat, lng, image_url, detail_url ของสุนัขที่สูญหายทุกตัว ผ่าน
เจ้าของสุนัขกดปุ่ม "แจ้งสุนัขสูญหาย" และปักหมุดตำแหน่งบนแผนที่ ระบบบันทึก lost_latitude, lost_longitude และตั้ง is_lost=True พร้อมแสดงข้อความสำเร็จ ผ่าน
ผู้ใช้ที่ไม่ใช่เจ้าของสุนัขพยายามแจ้งสูญหายสุนัขของคนอื่น ระบบแสดง error "คุณไม่มีสิทธิ์ในการแจ้งสูญหายสุนัขตัวนี้" และ redirect ผ่าน

ตารางที่ 5.14: ผลการทดสอบระบบค้นหาสุนัขด้วย AI

การทดสอบเงื่อนไขในการทดสอบผลการทดสอบ
ทดสอบการค้นหาสุนัขด้วย AI ผู้ใช้เข้าหน้า Matchdog (GET /matchdog/) ระบบแสดงหน้า Dropzone สำหรับอัปโหลดรูปภาพเพื่อค้นหา ผ่าน
ผู้ใช้อัปโหลดรูปภาพสุนัขที่ชัดเจน และกดค้นหา ระบบส่งรูปไปยัง AI API แล้วแสดงผลลัพธ์สุนัขที่คล้ายกันพร้อมคะแนน similarity ผ่าน
ผู้ใช้ส่ง POST request โดยไม่แนบรูปภาพ ระบบแสดงหน้า Matchdog ใหม่โดยไม่มีผลลัพธ์ (ไม่ crash) ผ่าน
ผู้ใช้ที่ยังไม่ Login พยายามเข้าหน้า Matchdog ระบบ redirect ไปหน้า Login ผ่าน

5.6 การทดสอบระบบขออุปการะสุนัข (Adoption)

ตารางที่ 5.15: ผลการทดสอบการส่งคำขออุปการะสุนัข

การทดสอบเงื่อนไขในการทดสอบผลการทดสอบ
ทดสอบการส่งคำขออุปการะ ผู้ใช้กดส่งคำขออุปการะสุนัขจากหน้า dog_detail (POST) พร้อมระบุเหตุผล ระบบสร้าง AdoptionRequest ใหม่ แสดงข้อความ "ส่งคำขออุปการะสำเร็จ" และ redirect ผ่าน
ผู้ใช้ที่เคยส่งคำขอแล้ว (status=PENDING) พยายามส่งคำขออีกครั้งสำหรับสุนัขตัวเดียวกัน ระบบแสดง warning "คุณได้ส่งคำขออุปการะสุนัขตัวนี้ไปแล้ว กรุณารอการตรวจสอบ" ผ่าน
ผู้ใช้ที่ยังไม่ Login พยายามส่งคำขออุปการะ ระบบ redirect ไปหน้า Login ผ่าน
ผู้ใช้ส่ง GET request ไปที่ URL ขออุปการะ (ไม่ผ่าน POST) ระบบ redirect กลับหน้า dog_detail โดยไม่สร้าง AdoptionRequest ผ่าน

ตารางที่ 5.16: ผลการทดสอบการจัดการคำขออุปการะ (Org Admin)

การทดสอบเงื่อนไขในการทดสอบผลการทดสอบ
ทดสอบการจัดการคำขออุปการะ Org Admin เข้าหน้ารายการคำขออุปการะ ระบบแสดงคำขอ (status=PENDING) ของสุนัขในองค์กรเท่านั้น ผ่าน
ผู้ใช้ทั่วไปพยายามเข้าหน้ารายการคำขออุปการะ ระบบแสดง error "คุณไม่มีสิทธิ์เข้าถึงหน้านี้" และ redirect ไปหน้า Home ผ่าน
Org Admin กดอนุมัติ (action=approve) คำขออุปการะ ระบบเปลี่ยน status เป็น APPROVED, สร้าง AdoptionParent และส่ง Notification แจ้งผู้ใช้ ผ่าน
Org Admin กดปฏิเสธ (action=reject) คำขออุปการะ ระบบเปลี่ยน status เป็น REJECTED และ redirect ไปหน้ารายการคำขอ ผ่าน
ผู้ใช้ทั่วไปพยายามส่ง request เพื่ออนุมัติ/ปฏิเสธคำขอ ระบบแสดง error "คุณไม่มีสิทธิ์ดำเนินการ" และ redirect ผ่าน
